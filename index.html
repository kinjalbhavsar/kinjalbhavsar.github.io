Who I am

I'm a software engineer based in the San Francisco Bay Area, currently at Google in Mountain View where I work on the mobile software stack. I've spent my career digging deep into systems — from embedded hardware to operating systems to the layers of software that make modern mobile devices work.

My academic roots are in computer architecture, OS design, RTOS, networking, and embedded systems — and those fundamentals still shape how I approach every problem. I believe great engineering is invisible: it just works, at scale, without drama.

02 / Experience

Where I've worked

Software Engineer – Jetpack XR SceneCore

Google · Mountain View, CA
2025 – Present

Core contributor to Jetpack XR SceneCore, Google's open-source spatial computing library for Android XR — the foundation for building immersive 3D applications on Android devices.

  • Serve as API reviewer and Kotlin expert for the SceneCore library — reviewing 250+ CLs across the team, enforcing API council standards, Kotlin idioms, and library design consistency.
  • Designed and implemented the entity lifecycle management system — introduced DisposedException, isDisposed, automatic cleanup via XrReferenceCleaner, and a safe Entity.dispose() API, eliminating a class of resource-leak bugs in spatial apps.
  • Owned ActivitySpace spatial continuity: implemented gravity-aligned unscaled activity space, FSM-gated origin updates, yaw preservation, and worldspace scale overrides — ensuring stable, predictable XR session placement.
  • Expanded the public SceneCore API surface: unrestricted InteractableComponent, InputEvent, GltfModelEntity; exposed Component.onAttach / onDetach; shipped Kotlin Flow + lifecycle-aware listener extensions.
  • Led Java → Kotlin migration of the SceneCore runtime — converting 20+ core classes including SpatialSceneRuntime, MovableComponentImpl, PanelEntityImpl, SpatialEnvironmentImpl, and more.
  • Modernized the SceneCore test app to MVVM + Jetpack Compose, replacing all XML layouts and adding ViewModel infrastructure — improving test coverage and developer velocity.
  • Fixed concurrency and correctness bugs: race condition in ActivitySpaceImpl origin updates, zero-matrix guard in system space pose, MovableComponent size propagation for SurfaceEntity.
Jetpack XR Android Spatial Computing Kotlin Jetpack Compose MVVM OpenXR AOSP

Software Engineer – ARCore

Google · Mountain View, CA
2019 – 2023

Worked on ARCore, Google's augmented reality platform powering AR experiences across hundreds of millions of Android devices worldwide.

  • Designed and launched camera and ARCore data source performance metrics — providing visibility into the camera pipeline's health and enabling data-driven optimization across devices.
  • Helped implement the Kotlin-based camera stack for ARCore, modernizing the core camera integration layer and improving developer ergonomics and maintainability.
  • Launched Electronic Image Stabilization (EIS) for ARCore — a gyroscope-driven stabilization feature that corrects rolling shutter and camera shake in real time, delivering smoother video and more stable AR overlays. Available across Android SDK (Kotlin/Java) and NDK (C). docs →
